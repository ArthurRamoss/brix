## Key Decisions Log

- **Monorepo over split repos**: single `git` history, single `.env.local`, `anchor test` + `pnpm --filter app` from root. Matches xVaultFi architecture reference.
- **LiteSVM over `anchor test` (Mocha)**: Anchor 1.0 scaffold defaults to Rust tests. Sub-second feedback loop on instruction logic; no TS overhead.
- **pnpm workspace at root**: app lockfile lives at repo root (`pnpm-lock.yaml`). `app/pnpm-workspace.yaml` removed after `create-next-app` ran.
- **Privy over Wallet Standard**: email-login UX critical for Selectimob landlord flow (non-crypto users). Phantom linking available as progressive upgrade for investors.
- **Repo on `/mnt/c/`**: accepted ~5x slower Anchor builds in exchange for single filesystem, simpler VS Code / Windows-side editing. Migrate to `~/brix/` only if build times become blocking.
- **Program ID placeholder**: `6xon...` is the scaffold-generated keypair. Will be re-generated or confirmed on first `anchor deploy --provider.cluster devnet`. Keypair at `target/deploy/brix-keypair.json` is **gitignored** — MUST back up before mainnet.

## Gotchas Noted

- Next.js 16 has breaking changes from training-data assumptions. Consult `app/node_modules/next/dist/docs/` before writing Next code.
- Privy pulls EVM deps (wagmi/reown/wallet-connect) transitively. Peer-dep warnings are non-blocking.
- `/mnt/c/` builds on WSL2 are slow due to 9P filesystem overhead — fine for MVP, migrate if painful.
